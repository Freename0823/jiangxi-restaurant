# 江西饭店独立站 · 填写说明

中日双语展示站 + 优惠券领取/核销后端。所有要替换的地方都用 `【】` 标记。

## 文件一览
- `index.html` — 顾客看的主页（含领券）
- `verify.html` — **店员核销页**（手机打开，勿外传）
- `schema.sql` — 后端数据库脚本（在 Supabase 跑一次）
- `supabase-config.js` — 填后端地址/key/优惠内容
- `images/` — 放菜品照片

## 怎么预览
主页 `index.html` 双击即可看外观。**但领券/核销功能要先配好后端**（见下），且需要把整个文件夹部署到网上（不能用 file:// 本地打开，相机和地址会受限）。

## 需要你替换的内容（搜 `【` 定位）
1. **关于我们** — 两段店铺故事（中/日各一份）。
2. **创业年份** — `【〇〇〇〇】`
3. **菜品** — 6 道菜已用江西名菜占位（瓦罐汤、藜蒿腊肉、拌粉、三杯鸡、米粉蒸肉、辣椒炒肉），可按实际菜单改名字和描述。
（店名、地址、电话、营业时间、交通、地图都已填好真实信息。）

## 优惠券后端配置（4 步）
1. 去 https://supabase.com 免费注册，新建一个 project。
2. 左侧 **SQL Editor** → 新建 query → 把 `schema.sql` 整段粘进去 → **Run**。
3. 左侧 **Project Settings → API**，复制 `Project URL` 和 `anon public` key，填进 `supabase-config.js`；同时把 `COUPON_OFFER` 改成你的实际优惠（如「全单9折」）。
4. 设置核销密码：回到 SQL Editor 跑一句（把密码换成自己的）：
   `update public.app_config set staff_pin = '你的核销密码' where id = 1;`

### 怎么用（主流程：顾客自助核销，店员不用操作）
- **顾客**：TikTok 简介挂主页链接（想统计渠道就用 `主页?src=tiktok`）→ 点「领取优惠券」→ 得到券码+二维码 → 到店点单时，**当着店员的面**点「点此使用」→ 屏幕变成「已使用」并显示一个**实时跳动的秒钟**。
- **店员**：只需瞄一眼顾客屏幕是不是「已使用」、时间在不在跳（截图是静止的，跳动的才是当场使用）。**不用扫码、不用密码。**
- 一张券只能使用一次，用过再打开永远显示「已使用」。

### 店员核销页 `verify.html`（后备 + 看数据）
平时用不到。两种场景才用：① 顾客手机不方便操作时，店员手动输码/扫码核销；② 想看活动数据。
打开 → 输核销 PIN（记住一次）→ 底部「刷新数据」看今日/累计领取与核销数。

### 关于「一人一码 + 没登录」
没有登录时，一人一码 = 一台设备一张券（存在浏览器里）。防重复用靠的是「每张码只能核销一次」+「已使用页面的实时跳动时钟」，截图蒙混不过去，无需登录。

## 加菜品照片
菜品现在是色块占位。想换成真实照片：把图片放进 `images/` 文件夹，然后把对应 `<div class="pic">...</div>` 改成：
```html
<div class="pic" style="background:none">
  <img src="images/瓦罐汤.jpg" alt="瓦罐煨汤" style="width:100%;height:100%;object-fit:cover">
</div>
```

## 语言切换
右上角「中文 / 日本語」按钮即可切换，会记住上次选择。默认中文。
